## 시뮬레이션 (Simulation)

### 개념

- 시뮬레이션(Simulation) 은 문제의 조건을 그대로 구현하면서, 실제 상황을 순서대로 모의 실험하는 알고리즘 기법이다.
- 수학적 공식이나 복잡한 알고리즘보다는, 문제의 규칙을 코드로 충실히 재현하는 것이 핵심이다.
- 보통 시뮬레이션 문제 = 구현력 문제라고 부른다.

### 예시: 로봇 이동 시뮬레이션

> 주어진 명령에 따라 로봇이 2차원 격자 위를 이동하는 과정을 그대로 구현한다.
> 
> 문제 설명
> 
> 로봇은 (0,0) 에서 시작한다.  
> 명령 문자열은 U, D, L, R로 구성된다. 
> U: 위로 한 칸  
> D: 아래로 한 칸  
> L: 왼쪽으로 한 칸  
> R: 오른쪽으로 한 칸  
> 모든 명령을 수행한 뒤의 좌표를 출력하라.

#### 구현 코드 (Java)
```java
public class SimulationExample {
public static void main(String[] args) {
String commands = "UURDDL";
int x = 0, y = 0; // 시작 좌표 (0,0)

        for (char c : commands.toCharArray()) {
            switch (c) {
                case 'U': y += 1; break;
                case 'D': y -= 1; break;
                case 'L': x -= 1; break;
                case 'R': x += 1; break;
            }
        }

        System.out.printf("최종 좌표: (%d, %d)%n", x, y);
    }
}
```

**출력**
```
최종 좌표: (0, 0)
```

### 알고리즘 문제 풀이 관점에서의 중요 포인트

| 구분       | 내용                                                                    |
|----------|-----------------------------------------------------------------------|
| 문제 해석력   | 시뮬레이션 문제는 문제의 조건을 정확히 구현하는 것이 전부이므로, 설명을 꼼꼼히 읽고 모든 규칙을 빠짐없이 코드로 옮겨야 함 |
| 자료 구조 선택 | 문제 상황에 맞는 자료 구조(배열, 큐, 스택, 맵, 리스트 등)를 상황 시뮬레이션용으로 선택해야 함              |
| 상태 관리    | 시뮬레이션은 ‘현재 상태 → 다음 상태’의 변화를 반복하므로, 위치, 방향, 점수 등 상태 변수를 명확히 정의해야 함     |
| 디버깅      | 단순한 로직이라도 조건이 많기 때문에, 중간 상태를 출력해 검증하는 습관이 중요                                                                      |


**시간 복잡도**  
대부분 O(N) 또는 O(N*M) 수준이지만, 경우의 수가 많아지면 단순 반복으로는 시간 초과 가능. 불필요한 연산을 줄이는 로직 필요

### 주 활용 사례

| 유형 | 예시 | 특징 |
|----|----|----|
| 격자 이동   | 로봇/청소기/탐사선 이동 문제   | 방향 벡터(dx, dy) 활용   |
| 게임 구현   | 뱀 게임, 테트리스, 2048 등   | 상태 갱신과 충돌 처리   |
| 시간 흐름 시뮬레이션   | 손님 입장, 대기열, 전염 확산   | 큐(Queue)와 단계별 업데이트   |
| 물리/자연 현상 모델링               | 불 퍼짐, 미세먼지 확산, 물 이동   | 2차원 배열 기반 단계별 변화   |
| 스케줄링/프로세스 시뮬레이션   | CPU 스케줄러, 교통 시뮬레이터   | 이벤트 순서대로 처리   |

### 한 줄 정리
> 시뮬레이션은 문제를 그대로 구현하는 사고력 테스트이며, **“조건 해석 + 상태 갱신 + 반복 구조”가 핵심**이다.  
> 복잡한 알고리즘보다 **정확한 구현력**이 중요하다.
